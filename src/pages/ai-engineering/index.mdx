This guide covers the engineering discipline of building, deploying, and operating AI systems in the real world. Where ml-essentials teaches you the mathematics and algorithms, and the agentic-ai series teaches you how to build autonomous agents, this series covers everything in between and underneath — the infrastructure, data systems, APIs, product patterns, and responsible practices that make AI work at scale.

The target reader is someone who understands ML fundamentals and wants to build production AI systems — not just run notebooks, but ship things that work reliably, efficiently, and responsibly.

The guide is organized into five parts, each covering a distinct engineering discipline.

> **This is a living roadmap.** Articles are written and published one by one. Items listed without a link are planned and coming. Linked items are published and ready to read.

## Capstone Projects

Three projects that integrate across the full engineering stack.

**Capstone 1: Build and Deploy a Production RAG System**
Build a complete RAG system from scratch: document ingestion pipeline, semantic chunking, embedding generation, vector DB indexing with hybrid search, re-ranking, LLM generation with structured output, evaluation framework with LLM judge, serving behind a FastAPI endpoint with dynamic batching, and a monitoring dashboard. Every component built and understood, not just wired together.

**Capstone 2: Fine-Tune and Serve an Open-Source LLM**
Fine-tune a 7B open-source model on a domain-specific task using QLoRA on a single GPU. Evaluate with a custom benchmark. Quantize to INT4 with GPTQ. Deploy with vLLM, add autoscaling, and instrument with full observability. Run a red-team evaluation and document findings in a model card. Measure cost-per-query and optimize to a target budget.

**Capstone 3: AI System Audit**
Take an existing AI system (a classifier, a RAG pipeline, or an LLM application) and perform a complete audit: bias evaluation across demographic subgroups, privacy risk assessment including a membership inference attack, red-team safety evaluation, performance profiling and optimization, and a full governance documentation package. The output is a written audit report with findings and recommendations.
