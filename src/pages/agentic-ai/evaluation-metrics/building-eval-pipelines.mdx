# Building Evaluation Pipelines

Modern AI systems evolve rapidly.

Agents are constantly updated with:

* new prompts
* improved reasoning strategies
* updated tools
* new model versions

Every change introduces the risk of **breaking existing behavior**.

To detect these issues early, teams build **evaluation pipelines**.

An evaluation pipeline automatically runs a large set of tests to measure agent performance.

Conceptually:

```text
Agent update
     ↓
Evaluation pipeline runs
     ↓
Performance metrics generated
     ↓
Regression detection
```

These pipelines function like **unit tests for AI systems**.

---

# Why Evaluation Pipelines Are Necessary

Without automated evaluation, improvements can unintentionally introduce new failures.

Example scenario:

```text
Agent version 1:
Correctly answers research questions.

Agent version 2:
Better reasoning but slower tool usage.

Agent version 3:
Faster execution but incorrect answers.
```

Evaluation pipelines detect these issues before deployment.

---

# Golden Datasets

A core component of evaluation pipelines is the **golden dataset**.

A golden dataset is a collection of tasks with **known correct outcomes**.

Example dataset entry:

```json
{
  "question": "Why do GPUs accelerate deep learning?",
  "expected_answer": "GPUs accelerate deep learning because they perform parallel matrix operations."
}
```

These datasets act as **reference benchmarks**.

---

# Characteristics of Good Golden Datasets

High-quality golden datasets have several properties.

| Property       | Description                  |
| -------------- | ---------------------------- |
| representative | covers real-world tasks      |
| diverse        | includes many task types     |
| verified       | answers validated by experts |
| stable         | remains consistent over time |

These properties ensure that evaluation results are meaningful.

---

# Example Golden Dataset Categories

Agent evaluation datasets often include multiple task categories.

Example categories:

| Category      | Example Task                     |
| ------------- | -------------------------------- |
| reasoning     | explain transformer architecture |
| retrieval     | find GPU benchmarks              |
| planning      | design a workflow                |
| tool usage    | query database                   |
| summarization | summarize research paper         |

This diversity ensures that agents are evaluated across **different capabilities**.

# Running Evaluation Tests

Evaluation pipelines run agents against the golden dataset.

Example workflow:

```text
Golden dataset
      ↓
Agent executes tasks
      ↓
Outputs recorded
      ↓
Evaluation metrics computed
```

Example result:

```text
Tasks evaluated: 500
Correct answers: 430
Success rate: 86%
```

This provides a quantitative measure of performance.

# Using LLM Judges in Pipelines

Many pipelines integrate **LLM-as-a-Judge** evaluation.

Example evaluation process:

```text
Agent generates answer
      ↓
Judge model evaluates answer
      ↓
Score recorded
```

Example judge output:

```json
{
  "score": 4,
  "reason": "Correct explanation but lacks detail."
}
```

These scores can be aggregated into performance metrics.

# Regression Testing for Agents

Regression testing ensures that new system updates do not break existing capabilities.

Conceptually:

```text
Previous agent version
      ↓
Evaluation baseline

New agent version
      ↓
Evaluation comparison
```

If the new version performs worse on certain tasks, a **regression is detected**.

---

# Example Regression Detection

Example evaluation comparison:

| Metric            | Old Version | New Version |
| ----------------- | ----------- | ----------- |
| task success      | 90%         | 85%         |
| reasoning quality | 4.2         | 4.1         |
| tool efficiency   | 3.9         | 4.3         |

Although tool efficiency improved, task success dropped.

This indicates a **potential regression**.

# Continuous Integration for Agents

Many organizations integrate evaluation pipelines into **continuous integration (CI)** systems.

Example workflow:

```text
Developer updates agent code
      ↓
CI pipeline triggers evaluation
      ↓
Tests run on golden dataset
      ↓
Results reported
```

If performance drops below a threshold, deployment may be blocked.

# Example Evaluation Script

<Tabs>

<TabItem label="Python" icon="seti:python">

```python
def run_evaluation(dataset, agent):

    results = []

    for task in dataset:

        answer = agent.solve(task["question"])

        results.append({
            "question": task["question"],
            "answer": answer
        })

    return results
```

<TabItem label="Rust" icon="seti:rust">

fn run_evaluation(dataset: &Vec<Task>, agent: &Agent) -> Vec<Result> {

    dataset.iter().map(|task| {

        let answer = agent.solve(&task.question);

        Result {
            question: task.question.clone(),
            answer
        }

    }).collect()
}
