# Building an MCP Server in Rust

The Model Context Protocol allows AI agents to interact with external systems through a standardized interface.

An **MCP server** acts as a gateway that exposes tools and resources.

Example architecture:

```text
AI Agent
   ↓
MCP Client
   ↓
MCP Server
   ↓
Tools / Databases / Filesystem
```

The server advertises available tools and executes requests sent by the agent.

In this article we will explore how to build a **simple MCP server in Rust**.

---

# Why Rust for MCP Servers?

Rust is an excellent language for building infrastructure services.

Key advantages include:

| Feature          | Benefit                             |
| ---------------- | ----------------------------------- |
| Memory safety    | prevents many runtime errors        |
| High performance | ideal for high-throughput systems   |
| Concurrency      | efficient handling of many requests |
| Reliability      | predictable behavior in production  |

Because MCP servers may handle:

* multiple concurrent tool calls
* large data transfers
* long-running agent workflows

Rust is a natural choice for implementing them.

# MCP Server Responsibilities

An MCP server typically performs several tasks.

```text
Register tools
      ↓
Advertise tool schemas
      ↓
Accept tool requests
      ↓
Execute tools
      ↓
Return results
```

The server must also expose **resources** such as files or databases.

# Designing Tool Interfaces

The first step is defining a **tool interface**.

Each tool should define:

* a name
* a description
* input parameters
* execution logic

Conceptually:

```text
Tool Definition
    ↓
Schema
    ↓
Execution Function
```

---

# Example Tool Interface

<Tabs>

<TabItem label="Rust" icon="seti:rust">

```rust
struct Tool {
    name: String,
    description: String,
}

trait ToolExecutor {
    fn execute(&self, args: serde_json::Value) -> serde_json::Value;
}
```

</TabItem>

<TabItem label="Python" icon="seti:python">

```python
class Tool:

    def __init__(self, name, description, handler):
        self.name = name
        self.description = description
        self.handler = handler

    def execute(self, args):
        return self.handler(**args)
```

</TabItem>

</Tabs>

This abstraction allows the server to **register multiple tools dynamically**.

# Registering Tools

# Exposing SQLite Databases

One common use case for MCP servers is exposing **databases to agents**.

SQLite is especially convenient because it requires **no external server**.

Example architecture:

```text
Agent
  ↓
MCP Server
  ↓
SQLite Database
```

---

# Example SQLite Tool

<Tabs>

<TabItem label="Rust" icon="seti:rust">

```rust
use rusqlite::{Connection, Result};

fn run_sql(query: &str) -> Result<Vec<String>> {

    let conn = Connection::open("data.db")?;

    let mut stmt = conn.prepare(query)?;

    let rows = stmt.query_map([], |row| row.get(0))?;

    Ok(rows.filter_map(Result::ok).collect())
}
```

</TabItem>

<TabItem label="Python" icon="seti:python">

```python
import sqlite3

def run_sql(query):

    conn = sqlite3.connect("data.db")

    cursor = conn.cursor()

    cursor.execute(query)

    return cursor.fetchall()
```

</TabItem>

</Tabs>

This allows agents to perform **database queries through MCP**.

# Handling Tool Requests

The MCP server receives requests from the agent and routes them to the appropriate tool.

Conceptual workflow:

```text
Request Received
      ↓
Parse Tool Name
      ↓
Lookup Tool
      ↓
Execute Tool
      ↓
Return Result
```

Example response:

```json
{
  "status": "success",
  "result": "query results"
}
```

---

# Example Tool Dispatcher

<Tabs>

<TabItem label="Rust" icon="seti:rust">

```rust
fn execute_tool(
    tools: &HashMap<String, Tool>,
    name: &str,
    args: serde_json::Value
) -> serde_json::Value {

    tools.get(name).unwrap().execute(args)
}
```

</TabItem>

<TabItem label="Python" icon="seti:python">

```python
def execute_tool(tools, name, args):

    return tools[name](**args)
```

</TabItem>

</Tabs>

This dispatcher allows the server to handle **dynamic tool calls**.

# Performance Advantages of Rust

Rust provides several advantages for MCP infrastructure.

### High Throughput

Rust servers can handle thousands of concurrent requests.

---

### Memory Safety

Rust prevents common issues such as:

* null pointer dereferences
* buffer overflows
* data races

---

### Efficient Concurrency

Rust supports async runtimes such as:

* Tokio
* async-std

Example:

```rust
#[tokio::main]
async fn main() {
    println!("MCP server running");
}
```

This enables efficient **parallel tool execution**.

# When Rust Is Particularly Valuable

Rust becomes especially useful when MCP servers must:

* serve many agents simultaneously
* execute heavy data processing
* stream large files
* run continuously in production environments

In these situations, Rust can significantly outperform dynamic languages.
