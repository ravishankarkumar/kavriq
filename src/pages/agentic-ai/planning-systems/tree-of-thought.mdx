# Tree-of-Thought and Algorithm-of-Thought

Chain-of-Thought reasoning introduced an important insight:

> LLMs perform better when they **explicitly reason through intermediate steps**.

However, Chain-of-Thought follows a **single linear path**.

```text
Problem
  ↓
Step 1
  ↓
Step 2
  ↓
Step 3
  ↓
Answer
```

If the reasoning goes wrong at any step, the final answer may also be incorrect.

Many problems require exploring **multiple possible reasoning paths**.

This idea led to the development of **Tree-of-Thought (ToT)** reasoning.

---

# Tree-of-Thought Reasoning

Tree-of-Thought extends Chain-of-Thought by allowing the model to explore **multiple reasoning branches**.

Conceptually:

```text
            Start
             |
      -----------------
      |               |
   Thought A      Thought B
      |               |
   Step A1         Step B1
      |               |
   Step A2         Step B2
```

Instead of committing to a single path, the agent evaluates several possibilities.

This approach resembles **search algorithms used in classical AI**.

---

# Why Multiple Reasoning Paths Matter

Consider the following problem:

> “Arrange the numbers 1–9 to form a magic square.”

A single reasoning path may fail.

However, exploring multiple possibilities increases the probability of success.

Tree-of-Thought enables the model to:

* propose multiple candidate steps
* evaluate their quality
* explore promising directions

This dramatically improves performance on **complex reasoning tasks**.

---

# The Tree-of-Thought Process

Tree-of-Thought reasoning typically follows four stages.

```text
Generate → Evaluate → Select → Expand
```

1. **Generate candidate thoughts**
2. **Evaluate the quality of each thought**
3. **Select promising branches**
4. **Expand the search tree**

This process continues until a solution is found.

---

# Example Tree-of-Thought Reasoning

Consider the task:

> “Find the best strategy to reduce energy consumption in a building.”

A Tree-of-Thought reasoning process might look like this:

```text
Thought A: Improve insulation
Thought B: Install solar panels
Thought C: Optimize HVAC system
```

Each branch can be expanded further:

```text
Improve insulation
   → wall insulation
   → roof insulation

Optimize HVAC
   → smart thermostats
   → heat recovery systems
```

The agent then evaluates which strategy is most promising.

---

# Evaluating Reasoning Paths

Tree-of-Thought systems typically evaluate candidate thoughts using the language model itself.

Example evaluation prompt:

```text
Rate the quality of the following reasoning step
on a scale from 1–10.
```

The model then selects the **best reasoning paths** to continue exploring.

This approach effectively turns the model into both:

* a **generator of ideas**
* a **critic of those ideas**

---

# Example Implementation

Below is a simplified Tree-of-Thought loop.

<Tabs>

<TabItem label="Python" icon="seti:python">

```python
def generate_thoughts(state, llm):

    prompt = f"""
    Given the current state:

    {state}

    Generate three possible next reasoning steps.
    """

    return llm.generate(prompt)
```

```python
def evaluate_thoughts(thoughts, llm):

    scores = []

    for thought in thoughts:

        prompt = f"Rate this reasoning step: {thought}"

        score = llm.generate(prompt)

        scores.append(score)

    return scores
```

</TabItem>

```rust
    let prompt = format!(

}
```

In practice, this process builds a **search tree of reasoning paths**.

# Search Strategies in Tree-of-Thought

Several search strategies can be used to explore the reasoning tree.

| Strategy             | Description                 |
| -------------------- | --------------------------- |
| Breadth-first search | explore all branches evenly |
| Depth-first search   | explore one branch deeply   |
| Beam search          | keep top-k best branches    |

Beam search is commonly used because it balances **exploration and efficiency**.

Example:

```text
Keep top 3 reasoning branches at each step
```

# Algorithm-of-Thought (AoT)

Algorithm-of-Thought takes the idea further.

Instead of free-form reasoning, the model follows a **structured algorithmic process**.

Example:

```text
Step 1: Identify relevant variables
Step 2: Apply formula
Step 3: Evaluate result
Step 4: Verify correctness
```

This encourages the model to behave more like a **program executing an algorithm**.

---

# Why Algorithm-of-Thought Works

Algorithm-of-Thought reduces ambiguity by enforcing **structured reasoning procedures**.

Benefits include:

* fewer reasoning errors
* more consistent outputs
* easier debugging

This approach is particularly useful for:

* mathematics
* programming tasks
* scientific reasoning

---

# Tree-of-Thought vs Chain-of-Thought

These methods differ in how they explore reasoning.

| Method           | Structure                |
| ---------------- | ------------------------ |
| Chain-of-Thought | single reasoning path    |
| Tree-of-Thought  | multiple reasoning paths |

Example:

Chain-of-Thought:

```text
Step 1 → Step 2 → Step 3
```

Tree-of-Thought:

```text
Step 1
 ├─ Step 2A
 ├─ Step 2B
 └─ Step 2C
```

Tree-of-Thought therefore provides **greater flexibility**.

---

# Limitations of Tree-of-Thought

Despite its power, Tree-of-Thought introduces new challenges.

### Computational Cost

Exploring multiple reasoning paths requires **more model calls**.

---

### Search Explosion

The number of branches may grow exponentially.

Example:

```text
3 branches per step
5 steps
= 243 reasoning paths
```

Efficient pruning strategies are therefore essential.

---

# Tree-of-Thought in Modern Agent Systems

Tree-of-Thought ideas influence many modern frameworks.

Examples include:

* reasoning models with **deliberate search**
* multi-agent debate systems
* planning graphs
* reinforcement learning planning systems

Although most production agents still use **ReAct-style loops**, Tree-of-Thought techniques are becoming increasingly important for **complex reasoning tasks**.

# From Trees to Graphs

One final observation:

Real-world planning problems often involve **revisiting states**.

This turns the reasoning structure from a tree into a **graph**.

Example:

```text
State A → State B → State C
      ↘
        State D
```
